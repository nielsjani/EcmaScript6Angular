Numbers

parseInt no longer converts binary or octagonal string representations of integers. Use Number instead

Number("someBinary")*

Binary value has to be prepended by 0b or 0B. Eg 0b101 is the binary for 5

Converted: {{result}}

Number("SomeOctagonal")*

Octagonal value has to be prepended by 0o or 0O. Eg 0o63 is the octagonal for 51

Converted: {{result}}

Number also has its own isFinite() and isNaN() methods. These almost behave the same way as the regular versions of these methods. The exception: both methods will always return false when passed a non integer value. Eg: Number.isFinite(7) returns true, Number.isFinite("7") returns false.

Number also gains the parseInt() and parseFloat() methods. These are equal to the old methods with the same name.

Number.isInteger(someInt)

Is 25 an integer? {{isInteger(25)}}

Is 25.456 an integer? {{isInteger(25.456)}}

Is 25.0 an integer? {{isInteger(25.0)}}

Is "25" an integer? {{isInteger("25")}}

Number.isSafeInteger(someInt)

Also: Number.MAX_SAFE_INTEGER and Number.MIN_SAFE_INTEGER

Is {{min-1}} a safe integer? {{isSafeInteger(min-1)}}

Is {{min}} a safe integer? {{isSafeInteger(min)}}

Is {{max}} a safe integer? {{isSafeInteger(max)}}

Is {{max+1}} a safe integer? {{isSafeInteger(max+1)}}
