let and hoisting

Variables declared in a function using the 'var' keyword, are automatically pulled up to the top of the function by the JS engine. This causes the variable to be accessible in the entire function (in the state "undefined" in places where you'd normally expect it to not be accessible.

Eg:

function getX(){

//JS engine will actually define foo on this line

if(something === true){

var foo = "x"

} else {

//foo is accessible here as "undefined"

}

}

The solution: the "let" keyword. It replaces the "var" keyword and all of its "odd" behaviour.

Eg:

Eg:

function getX(){

if(something === true){

let foo = "x"

} else {

//foo is not accessible here. Trying to do so, will result in an error

}

}

It is not impossible to define a "var" and "let" variable with the same name

Eg:

var x = "foo"

let x = "bar" //Will result in an error

TL:DR; Replace var with let and live a happier life!

const

const is a new keyword to define constand values. Eg: const DAYS_IN_WEEK = 7. const values always need to be initialized immediately. Just like let, const vars defined in functions only exists in block level scope (eg in an if, but not in the if's else).
